# HookahGuide — мобильное приложение

Клиент к API HookahGuide. Живёт в этом же монорепозитории, но **на сервер не
деплоится** — сервер собирает только бэкенд (Docker-сборка не включает `mobile/`,
см. корневой `.dockerignore`).

## Контекст для разработки

- **Контракт API:** [`../docs/MOBILE_API_SPEC.md`](../docs/MOBILE_API_SPEC.md) —
  все эндпоинты, модели, примеры ответов, экраны.
- **Базовый URL (прод):** `http://217.144.103.16/`
- **Как работает бэкенд:** скилл `.claude/skills/hookahguide-backend/`.

## Что внутри (заполняется в сессии разработки приложения)

Здесь будет код мобильного клиента (стек выбирается при старте: нативный
Android/iOS или кросс-платформа). Структуру и зависимости создаст отдельная сессия,
посвящённая приложению.

## Важно

- HTTP без TLS (пока нет домена) — в дев-сборке разрешить cleartext-трафик
  (Android `usesCleartextTraffic`, iOS ATS-исключение). Подробности — в спеке.
- Сборочные артефакты приложения игнорируются git (см. корневой `.gitignore`)
  и не входят в образ бэкенда.
